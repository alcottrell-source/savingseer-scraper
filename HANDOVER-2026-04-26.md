### Supabase schema migration (you ran these)
The scorer was rewritten to use the Tide Model, which writes different columns than the old Apple Model. You applied these migrations:
```sql
ALTER TABLE centre_seer_scores
  ADD COLUMN IF NOT EXISTS tide_score      numeric,
  ADD COLUMN IF NOT EXISTS bluf            text,
  ADD COLUMN IF NOT EXISTS brands_on_sale  int,
  ADD COLUMN IF NOT EXISTS total_brands    int;

ALTER TABLE centres ADD COLUMN IF NOT EXISTS tide_history jsonb;
```

### Tide history (sparkline data)
`score.js` now writes a `tide_history` JSON array to each row in the `centres` table after every scoring run. Contains up to 60 days of `{date, score}` pairs, oldest first. Ready to use as sparkline data in a frontend. Example:
```json
[
  { "date": "2026-04-23", "score": 48.0 },
  { "date": "2026-04-24", "score": 65.7 }
]
```
