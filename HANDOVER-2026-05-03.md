### 2. Score card

- Score reads `62/100` so the 0–100 scale is obvious at a glance
- **Phase gauge redesigned to 5 evenly-spaced stages** with `Go Now` exactly at the top centre:
  `Turning → Rising → Go Now → Falling → Low`
- Active stage label is bigger, bolder, in the stage colour. Inactive labels dim to 55% opacity
- Indicator: stage-coloured halo + white ring + filled centre dot — clearer "you are here" affordance
- Subtle drop shadow under the gradient stroke for depth
- **Removed the lower dual-arc personal-score widget** — wasn't ready to ship; will return in a separate redesign
- **Fixed broken arc geometry** — for any score >50 the path was telling SVG to draw the *long* arc through the bottom of the imaginary full circle, way outside the viewBox. Only the rounded line caps were visible (the floating wedges users were seeing). Bounded on a 180° gauge the arc is always ≤180°, so `largeArc` is always 0

### 3. History chart

- Now reads real 60-day `tide_history` from Supabase (was falling back to mock for everyone before the rewire)
- **Adaptive smoothing window** — `k = max(1, min(5, floor(n/8)))`. Short series (e.g. a centre with 10 days of history) no longer get flattened into one shallow curve; recent shape is preserved
- "Preview data — no live history yet" badge surfaces clearly when a centre's series is too short
- Trajectory label on the chart and on the score card now agree (single source of truth — passed in from `renderCentre`)

### 4. Brand list

- **One brand per row** (was 2-column, content was clipping on long names like River Island and & Other Stories)
- **Sorted by sale-status and freshness:**
  1. *On sale · newest first* — sorted by days running ascending
  2. *Not on sale* — alphabetical
  - Cluster groupings (High Street / Contemporary / etc.) dropped
- Each chip shows: favicon, brand name, **`N% off`** pill (when scraper has a clean number), days-running label, sale dot, click-through arrow `↗`
- **Every brand is tappable** — opens its `/sale` page in a new tab. Skimlinks rewrites at click time
- **Right-side meta is aligned across all rows** — fixed-width columns for `% off` (56px) and days label (72px, right-aligned). Empty placeholder spans where content is missing so dot/arrow x-positions never wobble
- **Days label is meaningful**, mapped to the freshness curve:
  - Day 1 → `New today`
  - Days 2–7 → `Fresh · Nd`
  - Days 8–21 → `Nd in`
  - Days 22+ → `Late · Nd`
- **`% off` plumbing fixed** — `max_discount_pct` now in the `brand_sale_events` SELECT, surfaced as an amber pill. Brands where the scraper can't extract a clean number stay silent (no fake "0% off")
- **Logos** wired from a `BRAND_URL_BY_NAME` map (mirrors `brands.js`, with aliases for dashboard name variants like `M&S`/`Marks & Spencer`). Source: Google favicon service. Quality is favicon-grade — see Limitations

### 5. Tide language

- All fruit / ripeness / "Apple Model" copy removed throughout
- Empty state: "When is the right moment to go?"
- Stage labels: `Flat / Turning / Rising / High Tide / Falling / Low` (Flat now dropped — see below)

### 6. Scorer (`score.js`) — server side

- Updated to match the new 5-stage gauge: dropped the `Flat` stage from `PHASE_NUMBER`, `getTideStage`, and the console icons map
- Empty centres (score 0) keep the user-friendly `Nothing on` verdict text but report stage `Turning` so the dashboard arc indicator lands at the cycle-start position
- **Takes effect on tomorrow's 08:00 UTC run** (GitHub Actions). Until then, today's `centre_seer_scores` rows for any zero-score centres still carry `stage: 'Flat' / verdict: 'Nothing on'` — the dashboard tolerates this because stage is computed locally; only the badge text would briefly lag

## Live data — what's real

Daily pipeline runs via GitHub Actions:
- **06:00 UTC** — scraper hits each brand's site, writes `brand_sale_events` (sale_status, date_first_detected, max_discount_pct)
- **08:00 UTC** — scorer computes today's centre score, writes `centre_seer_scores` and updates each centre's 60-day `tide_history` array

The dashboard reads all three tables via the Supabase anon role.

**Carry-over scraper issues** (not fixed this session):
- Next: 403 (bot protection)
- FLANNELS: HTTP/2 issue
- Both brands show as no-data until those scrapers are fixed

## Known limitations

1. **Logos are favicons, not brand logos.** Tried Brandfetch CDN first; it didn't deliver clean logos in production. Switched to Google's favicon service as a "ship now" fix. Quality is favicon-grade. **Upgrade path:** curated SVGs in `/public/logos/`, ~80 files, day's work
2. **Some brands have no URL / logo / clickthrough** — Primark, Uniqlo, AllSaints, Boux Avenue, Burton, Calvin Klein, Foot Locker, French Connection, Gant, Jack & Jones, Jaeger, Jigsaw, LK Bennett, Mint Velvet, Miss Selfridge are in the inline `BRANDS` list but not in `brands.js`/`BRAND_URL_BY_NAME`. Easy to add per brand
3. **`% off` coverage is patchy** — only shown when the scraper extracts a clean number from a brand's site. Many brands don't publish a clean percentage
4. **Personal-score visualisation parked** — old dual-arc widget removed; needs fresh design
5. **Dead code in `index.html`** — `buildArcSVG` and `arcPath` functions still defined but no longer called. Harmless, can clean up later
