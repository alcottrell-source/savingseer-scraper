## Score-card semantic fix — `claude/fix-tide-status-J1Dd6`

User flagged a screenshot of Festival Place, Basingstoke showing simultaneously:
- Score `54/100`
- Badge `HIGH TIDE`
- Trajectory `↑ Rising`
- Verdict `GO NOW` / *"Maximum density, maximum freshness. This is the moment."*

"Go now" implies peaked; "Rising" implies not yet peaked. The two contradict on the same card.

**Fix shipped (score-as-headline + sticky High Tide + demoted trajectory).**

1. **Threshold tightened.** High Tide stage now requires score `≥75` (was `≥50`). Score is the headline; trajectory only kicks in to distinguish Falling vs Low at the cycle end. Score 54 + Rising now reads as `Rising / Worth watching` — directly addresses the "54 doesn't feel like high tide" intuition.

2. **Sticky High Tide.** Once a centre crosses `≥75` it stays at High Tide for the next **5 days** even if today's score dips, as long as it hasn't crashed below `50`. This prevents the "Go now today, Last chance tomorrow" whiplash — a centre that hits peak deserves a stable window for users to actually act on it.
   - Tunables: `HIGH_TIDE_THRESHOLD = 75`, `STICKY_FLOOR = 50`, `RECENT_LOOKBACK_DAYS = 5` in `score.js`. Mirrored in `index.html`.
   - Lookback window for the scorer's recent-scores fetch extended from 3 days → 5 days. Trajectory still uses the first 3 entries, so its behaviour is unchanged.
   - Crash protection: a centre that drops from 80 → 45 in one day no longer says "Go now" — score below `STICKY_FLOOR` exits the sticky window.

3. **Trajectory demoted from competing badge to small forward-guidance copy** below the bluf:
   - `↑ Score rising — could climb higher tomorrow` (italic, muted)
   - `↓ Score dropping — moment is passing`
   - FLAT → no extra line (the bluf already conveys "holding").
   The `traj-indicator` block in the score card's top-right is gone. The history-chart still shows the small `↑ Rising / ↓ Falling / → Flat` arrow next to the *"Tide over N days"* heading — that's separate and unchanged.

4. **Dead code cleanup.** `trajectoryDuration` in `index.html` was only feeding the old `traj-indicator`; removed.

**Transient:** today's `centre_seer_scores` rows still carry the pre-fix `verdict` strings for any centre. Dashboard reads `verdict` from the server when present, so the badge text won't fully reflect the new stage logic until tomorrow's 08:00 UTC scorer run rewrites those rows. Same migration pattern as Flat→Turning earlier this week.
